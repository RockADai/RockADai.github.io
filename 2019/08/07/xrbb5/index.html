“仙人板板”STM32版04 | 结局与开局

发表于 2019-08-07 | 分类于 单片机开发
SC-STM终于有了进展，VR-STM也该开始了

  转眼一个月过去了才更新，果然人类的本质是真香啊。和之前等待毕业的咸鱼时期不一样，这一个月呆某可没闲着，适应公司工作，做公司的项目，熟悉公司环境，事情多得很。作为新员工，来到这个什么都很正式的大公司，是免不了一些所谓集中培训的活动的，没想到期间奇遇着实不少，现在就希望有几个兄弟在旁边啊，就着小酒，我可以把天都吹下来。

  那么，无限的心绪，就等有酒再叙，先继续项目的开发吧。

SC-STM的进展

  经过了好几次的设计修改和好几百元制板费的氪金，这次的SC-STM全测终于通过了。最终的设计和原来相比已是大相径庭，以CH340G为主的ISP功能因为更加便捷的ST-Link的引入而被砍掉了，节省出了好大一片的空间，让我得以将板子长度缩小了10mm，现在板子上的USB接口只有Slave功能了。除此之外，我采纳了我师父的意见，修改了很多过孔和线路，整个板子的布局都改变了。终于，测试长时间通电，板子表现稳定，测试程序烧写并运行，板子仍然表现稳定。目前计划就这么用着，等VR-STM的设计结束之后，会综合朋友们提出的意见最后对SC-STM整改一次，作为公测结束的标志。前方多图预警：

VR-STM启动

  VR即Various Resources，顾名思义，这次需要板载很多常见的外设资源。资源的选取上有三个原则，一是经久不衰，如CAN总线、485等；二是时尚流行，比如ESP8266和六轴加速度传感器，尽管呆某是个不爱赶时髦的音乐人，但在技术上，谁不赶时髦谁是傻X，没办法，科学在发展嘛；三是契合主控，比如说32位机就该配个电容屏，配个12864液晶那是屈才，当然下一步做8位机的时候就要配12864了。

  在大板上我选择的主控芯片是STM32F407ZGT6，相比于F1，F4的设计要复杂的多，但功能也要强大得多，所以即使有前车之鉴，这次做起来挑战仍然不小。

F4和F1的异同及处理方式

  因为大板对空间没有限制，所以这次把RTC电池加上了。手册要求VBAT不能空载，考虑到电池一般不添加，还要拉一路3.3V到VBAT上，所以用了一个BAT54C做电源选择（如上图）。

  143脚的定义是PDR_ON，这个脚是144脚以上封装才有的，支持STM32全系列，用来使能电压监视器和掉电复位，手册上建议F4系列拉高，F1系列拉低。此外还有两个引脚，VCAP_1和VCAP_2，这两个引脚和内部的电压调节器有关，手册要求接一个2.2uF电容拉到地。其余的引脚处理方式基本和F1相同。

调试

  ST-Link也并不是完美无缺的，实测发现它速度要略慢于JTAG，所以大板上把JTAG也加上了，反正不差空间嘛。

存储

  为了跑嵌入式系统，我在板子上放了SRAM和FLASH，后面还会放EEPROM来支持I2C总线。为了保险起见（省点重做板子的钱），这里以及后面绝大部分都参考了野火大神的设计：

SRAM和FLASH

  这部分理解起来比较简单，SRAM有19个地址位和16个数据位，和STM32的FSMC外设各个地址位和数据位分别对应连接。FSMC直译为灵活的静态存储控制器，它可以直接驱动SRAM、NOR和NAND，也正如其名，它不能控制动态存储器。不过在我们的板子上够用了，之后还会用它驱动LCD等设备，呆某目前对这个东西并不完全了解，总之知道FSMC是一种很好用的控制总线就对了。SRAM上还有片选、输出使能、写使能等低电平有效的使能脚，都可以用FSMC上相应的引脚一一对应。8个脚的FLASH就更好办了，把它的SPI接口连到F4的SPI1上，写保护和通讯使能直接拉高，搞定。

供电

  这个板子一共三种供电方式，分别是调试器供电、USB供电和DC-DC降压电源。由于昨天刚做完一个DC-DC，所以这部分真是驾轻就熟，继续多图预警：

USB保护

  这里首先参考RT7272的手册做了一个降压电路，用来把6~12V的直流输入降到5V，3.3V的电压还是通过AMS1117来提供，此外，当外部DC电源和USB同时连接时，用一个MOS管断开USB供电，防止灌电流损坏USB口。

总结

  本集中呆某做完了STM32F4的一些基本的外围电路，下集会做一些设备到板子上去，争取10月之前能完工。另外，之前说好的二级驱动库，Cortex-M3版本的已经传到了GitHub上：https://github.com/RockADai/XRBB
